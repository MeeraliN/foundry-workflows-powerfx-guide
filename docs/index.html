Why does plain text fail in some agent inputs?

Because the field expects a message collection, not a raw string.

If the node expects chat-shaped input, the value needs to look like a user message rather than plain text. That is why UserMessage(Local.PromptText) works where Local.PromptText alone fails.

When should I use UserMessage(...)?

Use it when the workflow field expects a message object or message collection for an agent conversation.

Use plain text only when the field wants a string. Use UserMessage(...) when the field wants chat history or a message-shaped payload.

What does Name isn't valid usually mean?

The value is missing a scope prefix or the variable name is wrong.

In Foundry workflows, most values come from System. or Local.. If you reference Name when the actual variable is Local.Name, the formula fails.

What does Type mismatch usually mean?

The field expects one shape and your formula produces another.

Start by identifying the field type first. Then inspect your formula result. If needed, convert or reshape the output using helpers such as Text(), Value(), or a message constructor.

What is the difference between System. and Local.?

System. exposes workflow-provided values. Local. exposes values created inside your workflow.

Examples: System.LastMessageText is built in. Local.CustomerName exists because your workflow stored it.

Why does my condition fail even though the formula looks logical?

Because the field only accepts a Boolean.

Condition nodes need something that evaluates to true or false. A text value like "help" is not a Boolean. A formula like StartsWith(System.LastMessageText, "help") is.

How do I work with structured output from an agent?

Save the agent output as structured JSON, then read its fields as a record when possible.

If the workflow already stores structured output for you, prefer reading fields directly. Use ParseJSON only when you truly have JSON text that still needs parsing.

Why does my loop fail?

Because the for-each source is not actually a table or collection.

If the source is a single string or record, the loop cannot iterate. Confirm that the source really contains multiple items and use table functions such as CountRows to inspect it.

How should I handle missing values?

Treat missing values as a separate case instead of letting them fail silently.

Use IsBlank to detect missing data and Coalesce to provide a safe fallback when appropriate.

When should I use Text() or Value()?

Use them when the field expects a different primitive type than your current value.

Text() turns numbers or dates into strings. Value() turns numeric text into a number when the target field expects numeric comparison or math.

What is the difference between JSON() and ParseJSON()?

JSON() creates JSON text. ParseJSON() reads JSON text back into a usable shape.

If you already have a record from the workflow, do not convert it to JSON text just to parse it again.

What should a beginner learn first?

Start with scope, strings, Booleans, conditions, and one saved variable flow.

Do not start with JSON, loops, or advanced agent chaining. First learn how a value moves through a workflow and how to tell what type a field expects.

Why are the official docs not enough for this gap?

Because the official material is helpful but fragmented across workflow docs, Power Fx docs, and scattered examples.

The missing piece is a workflow-first explanation of types, message collections, and UI-specific formula behavior.

What is the fastest way to debug a failing workflow expression?

Check scope first, then expected type, then actual output shape.

Do not rewrite the whole formula immediately. First ask: is the name valid, is the field expecting a Boolean or string, and is my value actually a table, record, or message collection?
